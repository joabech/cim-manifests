# ADI Kuiper Linux

This workspace contains the ADI Kuiper Linux build system — a Debian-based Linux distribution for Analog Devices hardware.

## Overview

Kuiper is built using a two-layer pipeline:
1. **Host side** — `kuiper/build-docker.sh` (runs as root via `sudo`)
2. **Container side** — Docker container with cross-compilation tools and 8 build stages

The build is entirely **shell-script-driven** with simple **bash configuration variables** (not kconfig).

## Build System

### How Kuiper Builds

```bash
make sdk-build          # Build with current config (30-60 min)
```

This runs:
1. Validates prerequisites (Docker, config file)
2. Builds a Docker image from `Dockerfile` with cross-compilation tools (QEMU, debootstrap, gcc-arm-*)
3. Runs container with `--privileged` (needed for chroot + loop devices)
4. Executes 8 numbered stages: bootstrap → locale → tweaks → desktop → ADI tools → boot → extras → export
5. Outputs: `kuiper/kuiper-volume/image_YYYY-MM-DD-ADI-Kuiper-Linux-<arch>.zip`

### Configuration

All configuration happens via **preset config files**:

```bash
make install-kuiper-basic-armhf     # 32-bit headless (RPI + Zynq + Arria10 + Cyclone5)
make install-kuiper-desktop-armhf   # 32-bit + XFCE desktop
make install-kuiper-full-armhf      # 32-bit + all ADI libraries & tools

make install-kuiper-basic-arm64     # 64-bit headless (RPI + ZynqMP + Versal)
make install-kuiper-desktop-arm64   # 64-bit + XFCE desktop
make install-kuiper-full-arm64      # 64-bit + all ADI libraries & tools
```

Each `make install-*` target copies a pre-prepared config file to `kuiper/config`.

You can switch configurations at any time — targets can be re-run as many times as needed.

### Config Variables

The `config` file (sourced as bash variables) controls:

| Variable | Options | Purpose |
|---|---|---|

| `CONFIG_DESKTOP` | `y` \| `n` | XFCE4 desktop + X11VNC |
| `CONFIG_LIBIIO` | `y` \| `n` | IIO device library |
| `CONFIG_PYADI` | `y` \| `n` | Python IIO interfaces |
| `CONFIG_LIBM2K` | `y` \| `n` | ADALM2000 support |
| `CONFIG_LIBAD9361_IIO` | `y` \| `n` | AD9361 multi-chip sync |
| `CONFIG_LIBAD9166_IIO` | `y` \| `n` | AD9166 calibration |
| `CONFIG_IIO_OSCILLOSCOPE` | `y` \| `n` | GTK+ IIO visualization |
| `CONFIG_IIO_FM_RADIO` | `y` \| `n` | FM radio example |
| `CONFIG_FRU_TOOLS` | `y` \| `n` | FMC FRU tools |
| `CONFIG_JESD_EYE_SCAN_GTK` | `y` \| `n` | JESD204 eye scan |
| `CONFIG_COLORIMETER` | `y` \| `n` | EVAL-CN0363-PMDZ app |
| `CONFIG_SCOPY` | `y` \| `n` | Oscilloscope & signal analysis |
| `CONFIG_GNURADIO` | `y` \| `n` | GNU Radio framework |
| `CONFIG_GRM2K` | `y` \| `n` | GNU Radio + ADALM2000 |
| `CONFIG_LINUX_SCRIPTS` | `y` \| `n` | ADI Linux scripts |
| `CONFIG_RPI_BOOT_FILES` | `y` \| `n` | Raspberry Pi boot files |
| `CONFIG_ARCH_ZYNQ` | `y` \| `n` | Xilinx Zynq (32-bit) |
| `CONFIG_ARCH_ARRIA10` | `y` \| `n` | Intel Arria 10 (32-bit) |
| `CONFIG_ARCH_CYCLONE5` | `y` \| `n` | Intel Cyclone V (32-bit) |
| `CONFIG_ARCH_ZYNQMP` | `y` \| `n` | Xilinx ZynqMP (64-bit) |
| `CONFIG_ARCH_VERSAL` | `y` \| `n` | Xilinx Versal (64-bit) |

## Common Workflows

### Build for Raspberry Pi (64-bit, basic)
```bash
make install-kuiper-basic-arm64
make sdk-build
```

### Build for Zynq + Desktop environment (32-bit)
```bash
make install-kuiper-desktop-armhf
make sdk-build
```

### Build everything for 64-bit (all libraries)
```bash
make install-kuiper-full-arm64
make sdk-build
```

### Switch configuration and rebuild
```bash
make install-kuiper-basic-arm64     # Switch to 64-bit basic
make sdk-build                       # Build
make install-kuiper-full-arm64      # Switch to 64-bit full
make sdk-build                       # Build again with different config
```

### Clean build artifacts
```bash
make sdk-clean
```

### Check prerequisites
```bash
make sdk-test
```

### Flash image to SD card
```bash
make sdk-flash          # Prints flashing instructions
```

## Understanding the Build Stages

The 8 stages run inside the Docker container in order:

| Stage | Purpose |
|---|---|
| **01.bootstrap** | Create ARM rootfs via debootstrap; configure Debian repositories |
| **02.set-locale-and-timezone** | Locale, timezone, and mandatory packages |
| **03.system-tweaks** | User accounts, SSH, network, USB automount |
| **04.configure-desktop-env** | XFCE4 desktop + X11VNC (only if `CONFIG_DESKTOP=y`) |
| **05.adi-tools** | Install ADI libraries based on `CONFIG_*` flags |
| **06.boot-partition** | Boot files for RPI, Zynq, ZynqMP, Arria10, Cyclone5, Versal |
| **07.extra-tweaks** | Optional custom script; RPI-specific packages |
| **08.export-stage** | Partition image; create `.img` file; zip it |

Each stage has sub-stages with individual `run.sh` files. Stages respect the `CONFIG_*` flags exported from `config`.

## Customization

### Run custom build script

Set `EXTRA_SCRIPT` to a script path (relative to `kuiper/`) in the config file:

```bash
EXTRA_SCRIPT=stages/07.extra-tweaks/01.extra-scripts/examples/extra-script-example.sh
```

### Add custom Kuiper target

If you need a non-standard configuration (e.g., custom architecture mix, disabled features), you can:

1. Create a new config file in `configs/kuiper-custom.config`
2. Add an install target to the Makefile
3. Or manually edit `kuiper/config` and run `make sdk-build`

### Preserve Docker container

Set `PRESERVE_CONTAINER=y` in the config to keep the Docker container after build (useful for debugging):

```bash
PRESERVE_CONTAINER=y
make sdk-build
```

The container name will be `debian_<DEBIAN_VERSION>_rootfs_container`.

## Troubleshooting

### "Docker not found"
```bash
make sdk-test          # Checks Docker
cim install os-deps   # Install Docker if missing
```

### "This script must be run as root"
The build script requires sudo. The Makefile wraps it, but if running directly:
```bash
cd kuiper && sudo ./build-docker.sh
```

### Build fails mid-way

Check the Docker container logs (if `PRESERVE_CONTAINER=y`):
```bash
docker logs debian_trixie_rootfs_container
```

Or check `kuiper/kuiper-volume/` for partial build artifacts and logs.

### Build takes too long

Expected times:
- **Basic config**: 30–40 min
- **Full config** (all libraries): 45–60 min
- First build may be slower (Docker image creation, debootstrap)

Subsequent builds reuse the Docker image, so they're faster.

## Notes

- Configuration uses **bash variable sourcing**, not kconfig
- Docker images and containers are cleaned up after each build (unless `PRESERVE_CONTAINER=y`)
- Build output is a **gzipped `.zip` containing a partitioned `.img` file** (not separate partitions)
- The `kuiper-volume/` directory must exist and be writable (mounted into the container)
